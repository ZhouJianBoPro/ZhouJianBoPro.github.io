**构造对象的方法及优缺点**
- 采用构造函数传参的方式(参数过多的话会导致构造函数十分臃肿)
- 采用javaBean的写法(构造过程中可能会被分调到几个调用中，可能存在线程安全问题)
- 采用builder模式构造(解决了线程安全问题)

```
public class PersonBuilder {

    private String name;
    private String sex;
    private Integer age;

    public String getName() {
        return name;
    }

    public String getSex() {
        return sex;
    }

    public Integer getAge() {
        return age;
    }

    /**
     * Builder工厂类
     */
    public static final class Builder {
        private String name;
        private String sex;
        private Integer age;

        public static Builder newBuilder() {
            return new Builder();
        }

        public Builder withName(String name) {
            this.name = name;
            return this;
        }

        public Builder withSex(String sex) {
            this.sex = sex;
            return this;
        }

        public Builder withAge(Integer age) {
            this.age = age;
            return this;
        }

        public PersonBuilder build() {
            PersonBuilder personBuilder = new PersonBuilder();
            personBuilder.name = this.name;
            personBuilder.sex = this.sex;
            personBuilder.age = this.age;
            return personBuilder;
        }

    }
}
```
